# Job Agent вҖ” Job Scraping & Ingestion Service

## 1. Project Overview

Job Agent automates the tedious process of monitoring job boards. It:

- **Scrapes** public job listing pages (LinkedIn-style) without requiring a login, using Playwright's headless Chromium browser.
- **Extracts** structured data: job title, company, location, URL, and full description (from the detail page).
- **Deduplicates** listings at the DB level using `source_url` as the unique key.
- **Stores** everything in PostgreSQL with a clean relational schema.
- **Exposes** the data via a FastAPI REST API, with an endpoint to trigger scraping on demand.

### Tech Stack

| Layer              | Technology                          |
| ------------------ | ----------------------------------- |

| Testing            | pytest + pytest-asyncio + aiosqlite |

## 5. Setup Instructions

### Step 1 вҖ” Clone the repository

```bash
git clone https://github.com/your-org/job-agent.git
cd job-agent
```

### Step 2 вҖ” Create a virtual environment

```bash
python -m venv .venv

# Activate it:
# macOS / Linux:
source .venv/bin/activate

# Windows (PowerShell):
.venv\Scripts\Activate.ps1
```

### Step 3 вҖ” Install Python dependencies

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### Step 4 вҖ” Install Playwright browsers

Playwright bundles its own browser binaries. Install Chromium (the only browser we use):

```bash
playwright install chromium
```

To also install the browser system dependencies (needed on Linux CI / Docker):

```bash
playwright install-deps chromium
```

Verify the installation:

```bash
python -c "from playwright.sync_api import sync_playwright; print('Playwright OK')"
```

### Step 5 вҖ” Configure environment variables

```bash
cp .env.example .env
```

Open `.env` and update at minimum:

```dotenv
DATABASE_URL=postgresql+asyncpg://postgres:yourpassword@localhost:5432/jobagent
```

## 14. Ethical Scraping Guidelines

This project is built for **educational and personal use**. Before deploying in any capacity:

1. **Read the robots.txt** вҖ” Check `https://www.linkedin.com/robots.txt` (or your target site's robots.txt) and honour the `Disallow` rules.
2. **Read the Terms of Service** вҖ” Many job platforms prohibit automated scraping. Review their ToS before use.
3. **Respect rate limits** вҖ” The `SCRAPER_DELAY_MS` setting exists for this reason. Do not set it to 0.
4. **Do not scrape personal data** вҖ” Only extract publicly listed job information (title, company, description). Do not scrape recruiter contact details or user profiles.
5. **Do not bypass authentication** вҖ” This project intentionally only scrapes public pages (no login). Do not modify it to log in and scrape private content.
6. **Use a descriptive User-Agent** вҖ” In production, identify your bot in the User-Agent string.
